# Phase 3: Auth & Onboarding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire existing auth UI to real Supabase auth, protect app routes with RequireAuth, and add a first-run onboarding screen that routes new users into their first upload.

**Architecture:** AuthProvider wraps the entire app tree in main.tsx. RequireAuth guards the AppShell layout route, protecting all app routes while keeping /, /login, /signup, and /onboarding public. New SignupPage flow navigates to /onboarding on success; OnboardingPage opens the existing UploadModal on CTA click and skips to /dashboard otherwise.

**Tech Stack:** React 19, React Router v7, Supabase JS v2, Vitest 4 + Testing Library, Tailwind CSS v4

```tsx
// frontend/src/components/RequireAuth.tsx
import { Navigate } from 'react-router-dom'
import { useAuth } from '../auth/authContext'

export function RequireAuth({ children }: { children: React.ReactElement }) {
  const { session, loading } = useAuth()

  if (loading) {
    return (
      <div className="min-h-screen bg-surface flex items-center justify-center" role="status" aria-label="Loading">
        <div className="w-8 h-8 border-2 border-primary/20 border-t-primary rounded-full animate-spin" />
      </div>
    )
  }

  if (!session) return <Navigate to="/login" replace />
  return children
}
```

```tsx
async function onSubmit(e: FormEvent) {
  e.preventDefault()
  setError(null)
  setLoading(true)

  try {
    const { data, error: signUpError } = await supabase.auth.signUp({
      email,
      password,
      options: {
        data: { full_name: name },
      },
    })

    if (signUpError) {
      setError(signUpError.message)
      return
    }

    if (data.session) {
      navigate('/onboarding')
    } else {
      // Email confirmation still enabled on remote — guide the user
      setError('Account created — please check your email to verify before signing in.')
    }
  } catch (err) {
    setError(err instanceof Error ? err.message : 'Sign-up failed.')
  } finally {
    setLoading(false)
  }
}
```
